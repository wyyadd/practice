# HCI 小组: 仿生书会梦见电子笔吗?
## Problem  
读万卷书，行万里路，对于大学生来说，旅游已经成为一种颇为流行的开阔视野的方式，然而，读万卷书反而变得不易。如今，有了来自网络上游戏、电影、追剧等的诱惑，或者面对更丰富多彩的大学生活，读书时间一再压缩。通过调查分析，大学生阅读缺失有一下几点原因：
1. 读书时间不足
2. 缺乏阅读兴趣
3. 阅读空间越来越少
4. 学生的学业负担过重
5. 阅读的选择类型  

不仅如此，目前有些书本过于冗长，动辄上千页，质量参差不齐，阅读时往往难以理解，掌握不了关键部分。尤其是英文书，遇到生僻词或专业术语时，常常要打断阅读去查词，影响同学们沉浸式的学习体验。  
同学们对阅读的兴趣因为种种书本问题更加缺失，因此我们迫切需要一种辅助我们阅读的新型人机交互方式，让阅读者更加高效地阅读，提高对阅读的兴趣。
## Target User
学生群体，研究人员
## User analysis
读者：处于信息时代的读者们不喜欢大段的文字，更倾向于利用浅显易懂的标题来决定是否深挖；习惯了英特网的用户也习惯了能够快速定位到有用信息的超链接，因此不喜欢线性的阅读过程；阅读大部分时候是孤单的过程，但读者们往往会在阅读过程中产生自己的思考，得到一些很想与他人分享的想法。  
## Solution 任务 Task
我们小组提出一下几点提升阅读体验的人机交互方案：
### 1. 生成浓缩的课本：对每个段落进行概括以决定要不要跳过这一段。  
__目标__：在阅读课本时，对于长段落会自动概括生成一段100词以内的概括文本。  
__前提条件__：课本文本为中文、英文两种语言  
__子目标__：概括文本需要体现文章大意，需要包涵核心知识点，需要符合语法规则与语义规则。  
__相关特点__：  
- 概括文本的生成可以采用预处理的方法：即在阅读前生成所有段落的概括文本  
- 概括文本的生成：如果选取两个以上的段落，也可以在短时间内生成概括文本  
- 在生成概括文本的同时，在原文高亮每一段文字的要点，可辅助用户决定在阅读原文时是否要跳过这一段。   

### 2. 生成知识图谱以方便我们快速定位所需的知识并知道其依赖。
通过知识图谱，不仅可以直观的表现出知识间的依赖关系，还可以迅速的定位到知识的位置。传统的书本阅读在定位知识时只能通过索引搜索，效率较低，而如果通过知识图谱的方法可以迅速定位。  
此特性可便于行动不方便的人士阅读书籍。  
### 3. 为生僻词注解免去手动查词典。
生僻字或不熟悉的单词往往是我们阅读中的一大障碍，理解偏差可能会使文意差之千里。传统阅读中我们往往需要翻阅词典和借助网络来理解，而系统若能将生僻字自动解释并借助知识图谱的方法做词意解释，可极大方便阅读。  
### 4. 提取书中出现的概念，并给出简洁的定义。
__目标__：在阅读课本时，能根据书中复杂的概念, 提出简介的定义, 方便理解。    
__前提条件__：我们的技术能够理解书中的概念  
__子目标__：提出简介定义时, 不能错误定义, 导致读者理解错误。  
__相关特点__：  
- 能够提取概念关键字
- 能够简化冗长语句  
### 5. 对于好的提炼，可以人工收藏，机器也会把它作为数据集不断学习。
__目标__：对于好的提炼,  用户可以人工收藏, 我们的机器将其作为优秀的总结进行学习  
__前提条件__：保证用户的隐私性  
__子目标__：筛选出人工收藏的优秀提炼  
__相关特点__：  
- 既能作为数据集学习, 用户的隐私性又得到保障
- 能够识别出优秀的提炼   
### 6. 读者可以在每一段的后边附上自己的评论或者心得，以便后来者参考。
__目标__：在阅读文本时，如果有一段是其他读者高频批注的，则这一段会在用户阅读时高亮  
__前提条件__：联网或保存了上一次联网的缓存  
__子目标__：  
- 高亮的高频标注需要是一段完整的的句子，具有完整的语义信息。  
- 高亮的高频评论需要满足含义清晰，简短有代表性  
 
__相关特点__：优于Kindle阅读器上的体验 
## Usability requirements
__Effectiveness__:用户应该能够正常的使用这些功能。  
__Efficiency__: 软件的功能应该尽可能集成到用户的阅读流程中，即阅读中用户可以轻而易举地访问他需要的功能。  
__Safety__：用户的文件安全应该被保障，即使用软件的各种功能时，不会对原文件造成不可逆的破坏；另外也不能造成硬件设施的不可逆损伤。  
__Utility__：用户应该能自由选择软件的各功能所应用到的对象：对于文本而言，用户应该能够轻松的选择字，单词，段落，章节，乃至全文本，并对其应用软件的功能。  
__Learnability__：应该存在向导来帮助用户体验一边所有功能，还应该有一份帮助文档放于易于找到的位置。  
__Memorability__：用户应该能够通过界面元素的视觉形状会想起其对应功能，各元素的名称应该简练且易懂。

## 用户评价：  
### 1、生成浓缩的课本：对每个段落进行概括以决定要不要跳过这一段。  
#### 优点：   
1）系统反应迅捷，用户等待时间低体验良好。  
2）概括文本的出现，让用户可以在短时间内完成对一本书本的速览、获得全段/全书的核心要点，提升阅读效率。  
3）系统高亮提示，给予用户选择的权利，阅读体验感提升，且阅读选择权仍掌握在用户手中。  
#### 缺点：  
1）在购买新书前需要一定时间预处理，才可使用此功能  
2）系统对于用户“可跳页”的高亮提示可能会影响用户的阅读纯净体验，后续建议把这个功能作为可选（或放置“开启/关闭”选项）  
3）在用户跳走一页后，无法方便的跳回原来的位置，后续建议增加在15分钟内跳回的功能。  
#### 提升方案：  
总体来说，是一种前所未有的阅读体验，提升方案如下：  
1）建议把这个功能作为可选（或放置“开启/关闭”选项），可以给希望沉浸读书的用户纯净的体验  
2）在用户跳走一页后，增加在15分钟内跳回的功能。  

### 2、知识图谱功能：
#### 优点：
1）这是一个十分新颖的功能，相信许多用户都会在看到它的时候产生兴趣。  
2）使用这个功能的话，可能不那么明显的，在阅读时不会注意到的知识间的关系，会被显式的展现出来，让学到的知识关联性更强。  
3）透过产生知识图谱，让知识的检索变得方便，这样普通的教材也可以在日后当作reference来使用。  
#### 不足：
1）软件中知识图谱和正常的阅读流程是分开的，如果想要切换的话，要先回到主页，然后在进入知识图谱界面，这很不方便。  
2）知识图谱界面一口气显示了所有的知识条目，让初次接触的用户不知从何下手。  
#### 改进：
将知识图谱融合到正常阅读的流程中，即在侧边浮现该段落中涉及的知识。  

### 3、生僻词注解功能：
#### 优点：
1）为不认识某个单词的人省去了手动翻阅词典的麻烦。  
2）简短的释义直接浮现在软件底部，不会弹出额外的对话框之类，也就不会打断阅读。  
3）收藏单词功能方便了外语学习者。  
#### 缺点：
1）有些时候读者就算不知道单词的意思也懒得查词典，这种时候，该功能无效。  
2）有些时候一些单词有着读者不知道的用法，但读者并未意识到这一点而不会主动点击查词。  
3）简短的翻译有时候并不能满足读者的需求，他们可能需要更完善的解释。  
#### 改进：
标记常见但却使用了不常见用法的单词以提醒读者。  
为注解功能添加一个查看详情的界面。

### 4、提取书中出现的概念，并给出简洁的定义
#### 优点：
1）在阅读课本时，能根据书中复杂的概念, 提出简介的定义, 方便理解。  
2）点击段落就可以自动生成定义，十分方便。  
3）对于长难定义，学习时可以借助此工具简化句子， 提高学习效率。  
#### 缺点
1）有的时候可能不够精准甚至产生错误的定义，误导读者。  
2）导致用户过度依赖此工具，用户对复杂定义的分析拆分能力降低。  
#### 改进
一方面不断优化我们的模型，使产生的定义更加精准，另一方面，引导用户尝试理解句子，锻炼用户个人能力。  

### 5、对于好的提炼，可以人工收藏，机器也会把它作为数据集不断学习
#### 优点：
1）为用户提供个性化收藏空间，方便用户重温学习。  
2）机器能根据用户的收藏进行学习，提供更加个性化的推荐。  
3）用户可以分享自己的收藏到社交空间，交流感悟。  
#### 缺点
1）机器将用户的收藏作为数据集进行学习时可能会产生隐私问题。  
2）用户的无效收藏可能会产生脏数据，不利于训练模型。  
#### 改进
既能作为数据集学习, 用户的隐私性又得到保障。  
针对用户的收藏，能够识别出优秀的提炼。  

### 6、读者可以在每一段的后边附上自己的评论或者心得，以便后来者参考。  
#### 优点：  
1）在阅读文本时，如果有一段是其他读者高频批注的，则这一段会在用户阅读时高亮，给人一种“多人共读”的阅读体验  
2）此功能下，用户可以更好地理解文章核心要点，熟悉文章金句。  
3）此功能下，阅读从单一变为大众，用户可以获得看视频”弹幕“的愉快体验。  
#### 缺点  
1）需要定期联网与其他用户的评论同步  
2）只有大众的观点可以推送给其他用户，独特犀利的评论心得较难。  
#### 提升方案：  
后台编辑可以针对一些小众的却细致认真的批注进行人工推送。
